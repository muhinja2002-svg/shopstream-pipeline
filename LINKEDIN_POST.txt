Just shipped a project I'm genuinely proud of — a real-time e-commerce analytics pipeline called ShopStream 🛒

Here's what it does end-to-end:

1️⃣ A Python producer simulates realistic user sessions (page view → add to cart → order placed) and streams them to Redpanda, a Kafka-compatible message broker running in Docker

2️⃣ A Python consumer reads from the stream, validates each event, and persists it to DuckDB — a fast embedded OLAP database

3️⃣ dbt transforms the raw JSON into clean, typed models with documented lineage and automated data quality tests

4️⃣ A Streamlit dashboard queries DuckDB live and displays a conversion funnel, events over time, and top SKUs by revenue — auto-refreshing every 10 seconds

A few things I'm especially happy with:
✅ Session-based event generation — SKU and price stay consistent across the whole funnel, so the data actually makes sense
✅ Consumer idempotency — a group_id means restarting the consumer doesn't duplicate data
✅ Real dbt models — staging layer parses the JSON, mart layer handles the aggregation, and tests catch bad data automatically
✅ A live dashboard that tells the full story

Tech stack: Python · Redpanda · DuckDB · dbt · Streamlit · Docker · Plotly

I built this to deepen my understanding of streaming pipelines, the ELT pattern, and how to connect data engineering work to something a business stakeholder would actually care about.

The full code is on GitHub — link in the comments. Happy to connect with anyone working in data engineering or hiring for entry-level roles 🙌
